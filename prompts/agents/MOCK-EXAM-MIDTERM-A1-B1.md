### Question types per section

- **Grammar:** MCQ (4 options), fill-in-the-blank (single word/short phrase), error-detection (which underlined part is wrong)
- **Vocabulary:** meaning MCQ (English → Arabic or vice versa), synonym/antonym MCQ, fill-blank in context. Tests USAGE in context, not raw translation.
- **Reading:** 2 fresh short passages per exam (A1: 100–160 words each; B1: 220–350 words each), written by you using vocabulary and grammar from units 1–4. Each passage followed by 5 questions: A1 = 3 MCQ + 2 True/False; B1 = 2 MCQ + 2 True/False + 1 True/False/Not Given. Reading questions test **inference and understanding**, not just literal recall.
- **Spelling:** focus on **highlighted/key vocabulary from readings** in units 1–4. Two question types mixed:
  - "Pick correct spelling" — MCQ with 4 spellings; 3 are common misspellings (transposed letters, double-letter omissions, vowel confusion, etc.), 1 is correct.
  - "Type the word that means ___" — fill_blank where student must spell the English word correctly given an Arabic meaning OR a short English definition / context sentence with the word blanked.
- **Writing:** one short prompt, requires student typing freely. Min word count enforced; full marks awarded automatically on submission with valid word count.

### Writing scoring (zero-API approach)

- Student types into a textarea with **live word count**.
- Submit button is **disabled** until word count ≥ minimum.
- On submit, server re-counts words (whitespace split, server is authoritative).
- If `word_count ≥ minimum` → **full 10 points awarded automatically**.
- The text is stored in `mock_exam_attempts.writing_response` (TEXT, nullable).
- Trainer can later override the writing score via the `manual_writing_score` column (nullable). If set, the submit RPC's idempotent re-call recomputes total accordingly. (Not needed for launch; just leave the column.)

### Question authoring quality rules

- All instructions in Arabic.
- Stems in English (for grammar/reading/vocab/spelling content).
- Options in English where the question is about English (grammar, vocab, spelling) — but options in Arabic where the question asks for the **meaning** of an English word.
- No trick questions. No double negatives.
- For MCQ: exactly 4 options, exactly 1 correct. Distractors must be plausible — same part of speech, similar length, no obvious "throwaway" options.
- For fill-blank: store an `acceptable_answers` array (case-insensitive, trim whitespace) — e.g. for an article fill, `["a"]` or `["an"]`. Multiple acceptable spellings/contractions if reasonable. **For spelling fill-blanks, only the correct spelling is accepted** (that's the whole point).
- For error-detection: store the index (0–3) of the underlined wrong part.
- **Do not copy existing exercises verbatim.** Read the content, understand what was taught, and write fresh questions that test the same concepts from a new angle.

### DIFFICULTY CALIBRATION RULES (mandatory — Ali asked for this explicitly)

The exam must **push students to study**. Soft questions defeat the purpose. Per section, aim for this implicit distribution (never labeled):

- **~25% Easy:** anchored to material the student definitely saw. Builds confidence, prevents early panic. Example (A1 grammar): "She ___ a teacher. (is/am/are/be)" — direct rule recall.
- **~50% Medium:** tests real understanding, not memorization. Example (A1 grammar): "My brother and I ___ from Riyadh. (am/is/are/be)" — requires recognizing compound subject.
- **~25% Hard:** exposes gaps the student needs to revise. Example (A1 grammar): "There ___ three books on the table, but there ___ one pen. (are/is — student must pick both)" — tests subject-verb agreement with there + plural/singular nuance.

**Distractor authoring — make them work for it:**

- Target **the specific mistakes Arabic-speaking learners actually make** in English:
  - Article confusion (omitting articles before professions, wrong article with vowel sounds)
  - he/she pronoun confusion (no gendered pronouns in some Arabic contexts)
  - is/are agreement with collective nouns
  - present simple vs present continuous overlap
  - false friends (sympathy ≠ تعاطف-only; actually ≠ فعلياً)
  - preposition collocations (depend ON, interested IN, good AT)
  - past simple vs present perfect (for B1)
- For vocab MCQ in context: distractors should be the SAME part of speech, plausible in the sentence shape, and ideally other words from the same unit (so the student can't eliminate by recognizing only one word).
- For spelling MCQ: misspellings target real patterns — double-letter omission (accommodation → acomodation), vowel confusion (separate → seperate), transposed letters (receive → recieve), silent-letter drop (knife → nife).
- For reading inference: at least one of the 5 questions per passage must require synthesis ("What is the main idea?", "Why does the author mention X?") — not just "What color was the car?".

**Specifically AVOID:**
- Throwaway distractors (clearly wrong, wrong part of speech, wrong topic).
- Questions answerable by skimming for matching keywords (test understanding, not search).
- Trick wording. Negative double-negatives. Ambiguous stems where two options are defensible.
- Repeating the same testing pattern across the section (variety in question phrasing is required).

**Self-check after authoring:** before inserting questions in the seed script, read them as if you were the student. If you can answer 80%+ of them in under 10 minutes without reading carefully, the questions are too easy — rewrite the medium and hard ones.

-- =============================================================
-- RPC: mock_exam_start(p_exam_code text)
-- Returns: attempt_id, started_at, expires_at, exam metadata, questions array
-- =============================================================

CREATE OR REPLACE FUNCTION public.mock_exam_start(p_exam_code text)
RETURNS jsonb
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_student_id  uuid := auth.uid();
  v_student     public.profiles%ROWTYPE;
  v_exam        public.mock_exams%ROWTYPE;
  v_attempt     public.mock_exam_attempts%ROWTYPE;
  v_now         timestamptz := now();
  v_expires     timestamptz;
  v_questions   jsonb;
BEGIN
  IF v_student_id IS NULL THEN
    RAISE EXCEPTION 'not_authenticated';
  END IF;

  SELECT * INTO v_student FROM public.profiles WHERE id = v_student_id;
  IF v_student.role <> 'student' THEN
    RAISE EXCEPTION 'not_a_student';
  END IF;

  SELECT * INTO v_exam FROM public.mock_exams WHERE code = p_exam_code AND is_active = true;
  IF v_exam.id IS NULL THEN
    RAISE EXCEPTION 'exam_not_found_or_inactive';
  END IF;

  -- Visibility gate: in 'preview' mode, only admins + test accounts can proceed
  IF v_exam.visibility = 'preview'
     AND v_student.role <> 'admin'
     AND COALESCE(v_student.is_test_account, false) = false THEN
    RAISE EXCEPTION 'exam_in_preview_mode';
  END IF;

  -- Eligibility: student must be at the exam's level
  IF v_student.level_id <> v_exam.level_id THEN
    RAISE EXCEPTION 'student_level_mismatch';
  END IF;

  -- Window check
  IF v_now < v_exam.open_at THEN
    RAISE EXCEPTION 'exam_not_open_yet';
  END IF;
  IF v_now > v_exam.close_at THEN
    RAISE EXCEPTION 'exam_closed';
  END IF;

  -- Existing attempt?
  SELECT * INTO v_attempt
    FROM public.mock_exam_attempts
    WHERE exam_id = v_exam.id AND student_id = v_student_id;

  IF v_attempt.id IS NOT NULL THEN
    -- If already submitted → reject (one attempt rule)
    IF v_attempt.is_submitted THEN
      RAISE EXCEPTION 'already_submitted';
    END IF;
    -- Else resume in-flight attempt
    v_expires := v_attempt.expires_at;
  ELSE
    v_expires := v_now + (v_exam.duration_minutes || ' minutes')::interval;
    -- Cap expiry at exam.close_at so a student starting 5 min before close doesn't get full duration
    IF v_expires > v_exam.close_at THEN
      v_expires := v_exam.close_at;
    END IF;

    INSERT INTO public.mock_exam_attempts(
      exam_id, student_id, started_at, expires_at, user_agent
    ) VALUES (
      v_exam.id, v_student_id, v_now, v_expires, current_setting('request.headers', true)::jsonb->>'user-agent'
    )
    RETURNING * INTO v_attempt;

    INSERT INTO public.mock_exam_audit_log(attempt_id, student_id, event, details)
    VALUES (v_attempt.id, v_student_id, 'start', jsonb_build_object('exam_code', p_exam_code));
  END IF;

  -- Fetch questions (no correct answers leaked to client!)
  SELECT jsonb_agg(
    jsonb_build_object(
      'id',                 q.id,
      'section',            q.section,
      'order_index',        q.order_index,
      'question_type',      q.question_type,
      'passage_group',      q.passage_group,
      'passage_text',       q.passage_text,
      'passage_title',      q.passage_title,
      'stem',               q.stem,
      'options',            q.options,
      'writing_min_words',  q.writing_min_words,
      'points',             q.points
    )
    ORDER BY
      CASE q.section
        WHEN 'reading' THEN 1
        WHEN 'vocabulary' THEN 2
        WHEN 'grammar' THEN 3
        WHEN 'writing' THEN 4
      END,
      q.order_index
  )
  INTO v_questions
  FROM public.mock_exam_questions q
  WHERE q.exam_id = v_exam.id;

  -- Also fetch any saved answers (for resume)
  RETURN jsonb_build_object(
    'attempt_id',         v_attempt.id,
    'started_at',         v_attempt.started_at,
    'expires_at',         v_attempt.expires_at,
    'exam', jsonb_build_object(
      'code',             v_exam.code,
      'title_ar',         v_exam.title_ar,
      'subtitle_ar',      v_exam.subtitle_ar,
      'duration_minutes', v_exam.duration_minutes,
      'pass_threshold',   v_exam.pass_threshold,
      'total_points',     v_exam.total_points,
      'min_writing_words',v_exam.min_writing_words
    ),
    'questions',          COALESCE(v_questions, '[]'::jsonb),
    'saved_answers',      (
      SELECT COALESCE(jsonb_agg(jsonb_build_object(
        'question_id', a.question_id,
        'selected_index', a.selected_index,
        'text_answer', a.text_answer
      )), '[]'::jsonb)
      FROM public.mock_exam_answers a
      WHERE a.attempt_id = v_attempt.id
    ),
    'writing_response',   v_attempt.writing_response,
    'server_now',         v_now
  );
END;
$$;

GRANT EXECUTE ON FUNCTION public.mock_exam_start(text) TO authenticated;

-- =============================================================
-- RPC: mock_exam_submit(p_attempt_id)
-- Idempotent. Computes score. Marks passed/failed. Returns full breakdown.
-- =============================================================

CREATE OR REPLACE FUNCTION public.mock_exam_submit(p_attempt_id uuid, p_auto boolean DEFAULT false)
RETURNS jsonb
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_attempt   public.mock_exam_attempts%ROWTYPE;
  v_exam      public.mock_exams%ROWTYPE;
  v_grammar   numeric := 0;
  v_reading   numeric := 0;
  v_vocab     numeric := 0;
  v_spelling  numeric := 0;
  v_writing   numeric := 0;
  v_total     numeric := 0;
  v_passed    boolean;
BEGIN
  IF auth.uid() IS NULL THEN RAISE EXCEPTION 'not_authenticated'; END IF;

  SELECT * INTO v_attempt FROM public.mock_exam_attempts WHERE id = p_attempt_id;
  IF v_attempt.id IS NULL THEN RAISE EXCEPTION 'attempt_not_found'; END IF;
  IF v_attempt.student_id <> auth.uid() THEN RAISE EXCEPTION 'attempt_not_yours'; END IF;

  -- Idempotency: if already submitted, just return the existing result
  IF v_attempt.is_submitted THEN
    RETURN jsonb_build_object(
      'attempt_id', v_attempt.id,
      'score_total', v_attempt.score_total,
      'score_grammar', v_attempt.score_grammar,
      'score_reading', v_attempt.score_reading,
      'score_vocabulary', v_attempt.score_vocabulary,
      'score_spelling', v_attempt.score_spelling,
      'score_writing', v_attempt.score_writing,
      'passed', v_attempt.passed,
      'pass_threshold', (SELECT pass_threshold FROM public.mock_exams WHERE id = v_attempt.exam_id),
      'idempotent', true
    );
  END IF;

  SELECT * INTO v_exam FROM public.mock_exams WHERE id = v_attempt.exam_id;

  -- Grade non-writing answers
  -- For each answered question, compute correctness and points
  UPDATE public.mock_exam_answers a
  SET
    is_correct = CASE
      WHEN q.question_type IN ('mcq','true_false','true_false_ng','error_detection')
        THEN a.selected_index IS NOT NULL AND a.selected_index = q.correct_index
      WHEN q.question_type = 'fill_blank'
        THEN a.text_answer IS NOT NULL
             AND EXISTS (
               SELECT 1 FROM jsonb_array_elements_text(q.acceptable_answers) AS aa(val)
               WHERE lower(trim(aa.val)) = lower(trim(a.text_answer))
             )
      ELSE NULL
    END,
    points_awarded = CASE
      WHEN q.question_type IN ('mcq','true_false','true_false_ng','error_detection')
        THEN CASE WHEN a.selected_index = q.correct_index THEN q.points ELSE 0 END
      WHEN q.question_type = 'fill_blank'
        THEN CASE WHEN EXISTS (
               SELECT 1 FROM jsonb_array_elements_text(q.acceptable_answers) AS aa(val)
               WHERE lower(trim(aa.val)) = lower(trim(coalesce(a.text_answer,'')))
             ) THEN q.points ELSE 0 END
      ELSE 0
    END
  FROM public.mock_exam_questions q
  WHERE q.id = a.question_id
    AND a.attempt_id = p_attempt_id;

  -- Sum by section
  SELECT
    COALESCE(SUM(CASE WHEN q.section='grammar'    THEN a.points_awarded END), 0),
    COALESCE(SUM(CASE WHEN q.section='reading'    THEN a.points_awarded END), 0),
    COALESCE(SUM(CASE WHEN q.section='vocabulary' THEN a.points_awarded END), 0),
    COALESCE(SUM(CASE WHEN q.section='spelling'   THEN a.points_awarded END), 0)
  INTO v_grammar, v_reading, v_vocab, v_spelling
  FROM public.mock_exam_answers a
  JOIN public.mock_exam_questions q ON q.id = a.question_id
  WHERE a.attempt_id = p_attempt_id;

  -- Writing: full marks if word count >= min, else proportional zero
  IF v_attempt.writing_word_count >= v_exam.min_writing_words THEN
    v_writing := (SELECT points FROM public.mock_exam_questions WHERE exam_id = v_exam.id AND section='writing' LIMIT 1);
  ELSE
    v_writing := 0;
  END IF;

  -- Honor manual override if present
  IF v_attempt.manual_writing_score IS NOT NULL THEN
    v_writing := v_attempt.manual_writing_score;
  END IF;

  v_total  := ROUND(v_grammar + v_reading + v_vocab + v_spelling + v_writing, 2);
  v_passed := v_total >= v_exam.pass_threshold;

  UPDATE public.mock_exam_attempts
  SET is_submitted = true,
      is_auto_submitted = p_auto,
      submitted_at = now(),
      score_grammar    = v_grammar,
      score_reading    = v_reading,
      score_vocabulary = v_vocab,
      score_spelling   = v_spelling,
      score_writing    = v_writing,
      score_total      = v_total,
      passed           = v_passed,
      updated_at       = now()
  WHERE id = p_attempt_id;

  INSERT INTO public.mock_exam_audit_log(attempt_id, student_id, event, details)
  VALUES (p_attempt_id, auth.uid(), CASE WHEN p_auto THEN 'auto_submit' ELSE 'submit' END,
          jsonb_build_object('score_total', v_total, 'passed', v_passed));

  RETURN jsonb_build_object(
    'attempt_id', p_attempt_id,
    'score_total', v_total,
    'score_grammar', v_grammar,
    'score_reading', v_reading,
    'score_vocabulary', v_vocab,
    'score_spelling', v_spelling,
    'score_writing', v_writing,
    'passed', v_passed,
    'pass_threshold', v_exam.pass_threshold,
    'idempotent', false
  );
END;
$$;

GRANT EXECUTE ON FUNCTION public.mock_exam_submit(uuid, boolean) TO authenticated;

### C.2 Author the A1 exam (34 questions + 1 writing = 35 items)

Theme: Cumulative practice for L1 units 1–4. Topics will reflect what was actually taught — confirmed by your content extraction.

Quality bar:
- **Grammar (10 Qs, 3.0 pts each = 30 pts):** Cover the 4 most important grammar points across L1 units 1–4 (as identified by content extraction). Mix MCQ + fill-blank + error-detection. Examples: `to be` forms, articles a/an/the, present simple agreement, possessive pronouns, question words. Apply the **Difficulty Calibration Rules** — ~3 easy, ~5 medium, ~2 hard. Distractors target Arabic-speaker mistakes.
- **Reading (10 Qs across 2 passages, 2.5 pts each = 25 pts):** Write 2 NEW short passages (100–160 words each) that:
  - Use vocabulary from L1 units 1–4 (sprinkle in 4–6 target words per passage)
  - Use grammar at L1 level (simple sentences, mostly present simple)
  - Have a clear topic from one of the unit themes
  - Are followed by 5 questions each: 3 MCQ + 2 True/False
  - At least 1 of the 5 questions per passage must test inference / main idea, not literal recall
- **Vocabulary (8 Qs, 2.5 pts each = 20 pts):** Pull from L1 units 1–4 vocab lists. Prefer words that appeared highlighted in readings. Mix question types: ~3 meaning MCQ, ~3 fill-blank in context, ~2 synonym/antonym (light at A1). Distractors must be the same part of speech and CEFR-appropriate (no L4 words as distractors for A1).
- **Spelling (6 Qs, 2.5 pts each = 15 pts):** Focus on **highlighted vocabulary from the readings of L1 units 1–4**. Mix two types:
  - **3 × "Pick correct spelling" (MCQ):** 4 spellings — 1 correct, 3 plausible misspellings (real Arabic-speaker error patterns: vowel confusion, double-letter omission, transposed letters).
  - **3 × "Type the word" (fill_blank):** stem in Arabic ("اكتبي الكلمة الإنجليزية اللي معناها: المنزل") OR English context with the word blanked ("She lives in a small ___________ in Riyadh.") — student types the correct English spelling. Accept ONLY the correct spelling in `acceptable_answers` (that's the whole point).
- **Writing (1 prompt, 10 pts):** One short prompt at A1 level on a familiar topic (introduce yourself, describe your family, describe your daily routine, etc.). Min 50 words. Example: "Write about your daily morning routine. What time do you wake up? What do you do first? Use at least 50 words."

### C.3 Author the B1 exam (38 questions + 1 writing = 39 items)

Same approach, B1 level:
- **Grammar (12 Qs, 2.5 pts each = 30 pts):** Cover B1-level points from L3 units 1–4: past simple/past continuous contrast, present perfect, comparatives/superlatives, modals, conditional type 1, etc. Mix all 3 grammar question types. ~3 easy, ~6 medium, ~3 hard.
- **Reading (10 Qs across 2 passages, 2.5 pts each = 25 pts):** 2 passages (220–350 words each) using B1 grammar (some past tense, present perfect, conditionals) and B1 vocab. Each followed by 5 questions: 2 MCQ + 2 True/False + 1 True/False/Not Given. At least 1 question per passage must require inference.
- **Vocabulary (10 Qs, 2.0 pts each = 20 pts):** L3 highlighted vocab + general unit vocab. Include 2–3 synonyms/antonyms, 3–4 fill-blank in context (testing collocation and usage, not raw translation).
- **Spelling (6 Qs, 2.5 pts each = 15 pts):** B1 highlighted vocabulary from L3 units 1–4. Same two types as A1 but harder words and richer contexts. Misspellings target real B1-level patterns (silent letters, irregular verb forms, double consonants in past tense).
- **Writing (1 prompt, 10 pts):** B1-level prompt requiring reasoning or comparison. Min 80 words. Example: "Some people prefer to study online; others prefer in a classroom. Which do you prefer and why? Write at least 80 words."

    try {
      const code = profile.level_number === 1 ? 'midterm-mock-a1' : 'midterm-mock-b1';

      }
      setAnswers(a);

      // Compute initial timeLeft from expires_at - server_now
      const ms = new Date(data.expires_at).getTime() - new Date(data.server_now).getTime();
      setTimeLeft(Math.max(0, Math.floor(ms / 1000)));
    } catch (e) {

          .eq('student_id', profile.id)

        navigate('/student/mock-exam', { replace: true });

      }
    }
